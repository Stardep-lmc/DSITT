## 2. 关键发现与修复

### 🔥 Bug 修复：NWD 公式错误

**初始实现（错误）**：
```python
# 使用 W₂² (Wasserstein 距离的平方)
nwd = exp(-W₂² / C²)    # C=4.0
```

**问题**：对归一化坐标 [0,1]，小目标的 W₂² 极小（约 1e-5），C=4 导致 exp 趋近于 1，loss 趋近于 0，梯度几乎消失。

**修正后（正确）**：
```python
# 使用 W₂ (Wasserstein 距离本身，即取根号)
nwd = exp(-W₂ / C)      # C=0.1
```

**原因**：
1. 原 NWD 论文（AAAI 2022）使用 `exp(-W₂/C)` 而非 `exp(-W₂²/C²)`
2. 原论文的 C 是针对像素坐标设计的（~12）；归一化坐标需要 C≈0.1
3. 使用 `sqrt(W₂²)` 而非直接用 `W₂²`，保证距离度量的几何意义正确

### 验证结果对比

| 场景 | NWD loss | NWD grad | GIoU loss | GIoU grad |
|------|---------|----------|-----------|-----------|
| 极小目标 (0.01×0.01), 0.5% 偏移 | 0.068 | 9.32 | 1.079 | 90.39 |

**分析**：
- NWD 梯度约为 GIoU 的 1/10，但更加**稳定**和**平滑**
- GIoU 对小目标的 90 倍梯度会导致训练不稳定
- NWD 的梯度与目标大小成正比，大小目标获得均衡的学习信号

## 3. 训练验证结果

### Dummy 数据 2 epoch 训练对比

| 配置 | Epoch 1 Avg Loss | Epoch 2 Avg Loss | Box Loss 下降 |
|------|-----------------|-----------------|---------------|
| GIoU (baseline) | 6.06 | 2.55 | giou: 1.0→1.1 |
| **NWD** | 10.82 | 0.88 | nwd: 0.97→0.18 |

**观察**：
- NWD 模式的总 loss 在 epoch 2 降到 0.88，低于 GIoU 的 2.55
- NWD box loss 从 0.97 快速降到 0.18（81% 降幅），说明模型正在有效学习
- GIoU box loss 几乎不下降（1.0→1.1），说明 GIoU 对 dummy 小目标的优化效率低

## 4. 设计决策

### 4.1 保持 GIoU 向后兼容
- 通过 `box_loss_type` 参数在 GIoU 和 NWD 之间切换
- 匹配代价同步切换，保证损失和匹配的一致性
- 消融实验可以方便地对比两种模式

### 4.2 NWD 常数 C 的选择
- **C=0.1**：适合归一化 [0,1] 坐标（默认）
- 后续可搜索 C ∈ {0.05, 0.1, 0.2} 找最优值
- 如果使用像素坐标，C 应设为 ~12

### 4.3 NWD 在匹配中的意义
- NWD 匹配代价对小目标更友好：不会因为 IoU 为 0 而完全失去匹配信号
- 即使预测框和真实框不重叠，NWD 仍然提供有意义的距离度量
- 这对极小目标（<8px）的初始训练阶段尤为重要

## 5. 对 V2 路线图的影响

### 确认的收益
- NWD 确实为小目标提供了更好的训练信号
- 实现简单，无额外参数量开销
- 与 TALA 匹配自然集成

### 发现的新问题
- NWD 初始 loss 较高（88.6 vs GIoU 的 31.5），需要注意学习率的适配
- 可能需要在训练前期使用更小的 NWD 权重，后期增大
- C 值的最优选择需要在真实数据上验证

### 对后续阶段的建议
- 阶段二（MTUQ）中的跨模态一致性损失可以参考 NWD 的平滑特性
- 阶段四（SAS）的尺度自适应采样与 NWD 天然互补——NWD 提供尺度感知的损失，SAS 提供尺度感知的注意力
